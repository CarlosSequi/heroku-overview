Servicios incluidos en PaaS: Heroku



Adrián Morente Gabaldón

Aunque ya lo conocemos, vamos a hablar de uno de los PaaS más conocidos para el usuario novato. Comentaremos cuáles son los principales servicios interesantes para los desarrolladores y cuáles son las principales vías de negocio para la empresa que mantiene dicho PaaS (Salesforce).

¿Qué es Heroku?


  • Plataforma cloud para construir, desplegar, monitorizar y desplegar aplicaciones.
  • Planes Dynos de distinto volumen (y precio).
  • Herramientas GUI y CLI para desarrollo y gestión.
  • Soporte para equipos y empresas.
- Dynos: contenedores ligeros preparados para lenguajes específicos que buscan maximizar la productividad del desarrollador. - Como sabemos, en cloud todo consiste en pagar según utilización. - La herramienta CLI permite monitorizar y gestionar máquinas en ejecución (por ejemplo). - Existen planes desde precio gratuito hasta profesionales de gran escala (que incorporan cosas como escalado automático).

Experiencia del desarrollador (DX)

Servicios incluidos

  • Desarrollo de aplicaciones
  • Colaboración
  • Manejo de datos
  • Instrumentación, prestaciones y pruebas
  • Almacenamiento
  • Gestión de transacciones
Estos son los siguientes apartados que vamos a contemplar, como dice en las diapositivas de teoría. Empecemos con el desarrollo de aplicaciones.

Desarrollo de aplicaciones

Dev Center
  • Guía rápida para distintos lenguajes (usa Procfile).
  • Integración continua y despliegue automático desde CLI o Git.
  • Despliegue con imágenes de Docker (propias o de Container Registry.
  • Instrumentación con add-ons.

- Posee distintas guías para arranque rápido con distintos lenguajes (Procfile). - Integración continua (testing, despliegue automático). - Despliegue: con Docker "heroku.yml", con CLI (heroku ps).

Desarrollo - Pipelines

- Resume las distintas fases por las que DEBE pasar un proyecto (desarrollo, review, pruebas y producción). - La idea es tener diferentes instancias de distintas versiones de la aplicación.

Colaboración

  • Proyecto simple: permisos de acceso.
  • Equipos: gratis (<5 miembros), de pago (6-25 miembros, 10$/mes).
  • Enterprise: equipos más grandes, con roles, espacios privados, etc.

En equipos, los dynos gratuitos no están disponibles, solo las instancias de pago.

- El proyecto simple con permisos de acceso funciona como cualquier repositorio normal de Github. - Los equipos se rigen también por permisos y permiten compartir proyectos. - Enterprise es opción para equipos mucho más numerosos y con distintos roles.

Heroku Teams

Un equipo consta de administradores y miembros (con distintos permisos).

Manejo de datos

  • Heroku Postgres: servicio DDBB relacional (SQL).
  • Heroku Redis: almacén clave-valor (in-memory).
  • Apache Kafka on Heroku: envío de mensajes en "tiempo real". Patrón Pub/Sub.

Servicios ajenos a Heroku pero provistos de forma cómoda al desarrollador.

- Postgres: servicio SQL comúnmente conocido. - Redis. servicio mantenido en memoria principal, alta velocidad. Ambos son servicios ya conocidos y externos a Heroku pero Heroku te provee con una forma de instalar estos servicios de forma cómoda y tenerlos accesibles para acceder desde cualquier lenguaje. - Kafka: patrón Publish-Subscribe para streaming de datos en tiempo real.

Monitorización

El dashboard permite supervisar y gestionar los distintos servicios.

Reúne los logs, las diferentes instancias contratadas (cada una con su estado), etc.

Prestaciones

Heroku permite distintos tipos de ajuste del rendimiento necesario:

  • Escalado (manual o automático).
  • HTTP caching (a través de CDN).
  • Workers y trabajo en segundo plano.
- Escalado automático solo permitido en los dynos más caros. - Se cachean los assets estáticos para brindar más velocidad. No soportado sin configuración por parte de Heroku. - Trabajos en segundo plano como consultas a API externa, redimensión de imágenes, etc.
